## 要件定義

### 開発の背景と目的  

#### 開発背景

- 今までに自分のポートフォリオを作ったことがなく、いちから作ってみたかった。

#### 開発目的

- 自分が作ったもの（制作物）をまとめたい。

### 機能要件

<p>本システムで実現する機能やアプローチする課題を以下に示す。</p>

#### プロフィール

- プロフィール画像を載せる。
- GitHubとSNS（TwitterやInstagram）にアクセスを可能にする。
- 自己紹介などを載せる。

#### 制作物

- 制作物をまとめて、アクセスを可能にする。
- （可能であれば）制作物の詳細を記したページを作る。

#### 管理者用機能

- ポートフォリオに掲載する情報を投稿・編集するために、特定のユーザー（管理者）のみがアクセスできるページを作成する。詳しくは、以下の通りにする。
  - ログイン機能の実装。セキュリティ対策が目的。二段階認証を取り入れ、メールアドレスとパスワードによるログインの他、QRコードを用いる認証を行う。
  - 掲載する情報を投稿・編集するために、以下のページを作成する。

    | 情報の項目名 | 実施するモード | 目的                                         | 備考                                                                 |
    | ------------ | -------------- | -------------------------------------------- | -------------------------------------------------------------------- |
    | プロフィール | 編集           | プロフィールの編集                           | 設定するプロフィールは一人分のみであるため、投稿画面は作成しない。   |
    | 実績         | 投稿、編集     | 実績の投稿、編集                             |                                                                      |
    | お知らせ     | 投稿           | お知らせの投稿                               | 一度投稿したお知らせを編集する必要はないため、編集画面は作成しない。 |
    | 管理者設定   | 変更           | ログイン用のメールアドレスとパスワードの変更 |                                                                      |

#### その他

- バージョンアップをした際などに、それを「お知らせ」として、確認できるようにする。

<p>※バージョンは[基本設計書](./basic_design_document.md)で明記。</p>

| データベースサーバー | MySQL                      |
